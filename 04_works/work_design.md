# ワーク設計 — The Launch Code 5Days

> Phase 4 成果物
> 設計方針: 台本の流れを壊さず「後付けで差し込む」。ユーザーが自分で気づくことを最優先。

---

## 強めたい4要素と対象Day

| 要素 | 対象Day | 目的 |
|------|--------|------|
| 現在の状況の明確化 | Day1 | 「自分のスキルが商品になると気づいていない」現状を認識させる |
| 理想の未来の明確化 | Day1〜2 | 「どこに行きたいか」をリアルに言語化させる |
| 課題の明確化 | Day2〜3 | 「設計図がない」「一人では回せない」に自分でたどり着かせる |
| 課題解決後のイメージ | Day4 | 「穴を埋めた先の未来」を書いてセールスへ接続 |

## Day1 ワーク設計

### ワーク① スキル棚卸し（台本既存・ヘビーワーク）

> 台本PART7（1:35〜1:45）に既に組み込み済み。確認・詳細化のみ。

**目的**: 現在の状況の明確化 × 理想の未来への足がかり

**強める要素**: 現在の状況の明確化

**タイプ**: ワーク型（手を動かす）

**所要時間**: 10分

**ファシリテーション**:
```
「"自分が当たり前にできること"を10個書き出してください。
 すごいことじゃなくていい。仕事でやってること、人から頼まれること、
 なんとなく得意なこと、全部OK。
 制限時間は5分。いっぱい出てきた人は絞らなくていいです。どんどん書いて。」

（5分後）

「書き出したもの、チャットで一部でいいので見せてもらえますか？
 あ、これ全部。全部商品になれるやつですよ。
 後で詳しく話しますが、"当たり前"こそが一番売れる。」
```

**オプチャ課題**: 書き出した10個をオプチャに投稿（任意）

**設計意図**:
- 「自分には何もない」という現状認識を崩す第一歩
- 書き出す行為で内的動機づけが発動（自分で出したものは否定しにくい）
- オプチャ投稿で他者の目線が入り「意外と価値ある」気づきが深まる

---

### ワーク② 理想の未来を一文で書く（新規追加・ライトワーク）

> Day1終盤、ワーク①の後に追加。30秒程度の軽い問いかけ。

**目的**: 理想の未来の明確化

**強める要素**: 理想の未来の明確化

**タイプ**: 問いかけ型（思考コスト低）

**所要時間**: 2〜3分

**ファシリテーション**:
```
「最後に一個だけ。30秒でいいです。

 "1年後、このスキルを使ってどうなっていたいか"、
 一文で書いてみてください。
 完璧じゃなくていい。思ったままで。」

（30〜60秒）

「書けた人、チャットに流してみてください。
 このビジョン、5日間持ち続けてください。
 Day5が終わった時、この一文を見返してほしい。」
```

**設計意図**:
- 5日間を通じた「軸」を最初に設定する
- Day5で回収できる伏線になる（「あの時書いた一文、変わりましたか？」）
- 内的動機づけを早期に発動させる

## Day2 ワーク設計

### ワーク③ 商品化3要素の言語化（台本既存・ヘビーワーク）

> 台本PART8（1:45〜1:50）に既に組み込み済み。詳細化。

**目的**: 課題の明確化（「設計図がない」の手前で自分のケースを具体化）

**強める要素**: 理想の未来の明確化 × 課題の明確化

**タイプ**: ワーク型

**所要時間**: 10分（記入5分 + チャット共有5分）

**3要素フォーム**:
```
① ターゲット（誰のために？）:
② スキル/サービス（何を提供する？）:
③ 変化/ベネフィット（どう変わる？）:

→ まとめた一文: 「私は（①）が（③）できるように（②）を提供する」
```

**ファシリテーション**:
```
「昨日書き出したスキル棚卸し、手元にありますか？
 今日の知識を使って、それを"商品の形"にしてみます。

 この3つを埋めてみてください。5分でできます。
 完璧に埋まらなくていい。空白があってもOK。
 それが今日の発見です。」

（5分後）

「空白があった人、手挙げてみてください。
 そこが"方法論の壁"です。明日それを壊しにいきます。」
```

**設計意図**:
- 「分かった」→「自分には当てはめられない」のギャップを自覚させる
- 空白を可視化することで、Day3への必然性を自分で感じる

## Day3 ワーク設計

### ワーク④ AIワーク①② 商品コンセプト＆LP生成（台本既存・体験型ヘビーワーク）

> 台本PART3〜4（0:45〜1:08）に既に組み込み済み。

**目的**: 課題解決の疑似体験 × 「できそう！」の高揚を最大化

**強める要素**: 課題の明確化（できるかも、という疑似成功体験）

**タイプ**: ワーク型（実際にAIを動かす）

**所要時間**: 約23分（ワーク①15分 + ワーク②8分）

**設計のポイント（追加意識）**:
- このワークの高揚感をDay4の梯子落としの「高さ」にする
- 「AIでここまでできた！一人でいける！」という感覚を意図的に作る
- 相互値付けも含め「でも本当に？」という布石を打っておく

**Day3末尾に追加する問いかけ（新規）**:
```
「今日AI使って商品コンセプトとLPのドラフトができましたよね。
 最後に一個だけ聞かせてください。

 "これ、一人で本番まで走りきれそう？"
 正直なところをチャットに流してみてください。」
```

**設計意図**:
- Day4の梯子落としへの「予告」を参加者自身の口から引き出す
- 「やれる気がするけど...不安」というリアルな反応を顕在化させる

## Day4 ワーク設計

### ワーク⑤ 一人で埋められない穴の書き出し（台本既存・最重要ヘビーワーク）

> 台本第2幕（0:40〜0:50）に既に組み込み済み。最も重要なワーク。

**目的**: 課題の明確化 × 課題解決後のイメージへの橋渡し

**強める要素**: 課題の明確化（環境が必要 = 自分で気づく）

**タイプ**: ワーク型（最重要）

**所要時間**: 10分

**ファシリテーション詳細**:
```
「ちょっとワークをしてほしいです。

 今、頭の中にローンチの全工程が見えてますよね。
 商品設計 → LP → 集客 → 教育コンテンツ → セールス → クロージング。

 この中で、"自分一人でやろうとしたら詰まるな"という工程、
 "埋められない穴"を書き出してみてください。

 正直に、全部書いていい。
 誰も責めないし、できない工程があっても当然です。

 5分、集中してください。」

（5分後）

「書き出した穴、いくつあった人が多いですか？
 1個? 2個? 3個以上?

（チャット見る）

 見てください。みんな同じ穴を持ってる。
 これ、個人の能力の問題じゃないんです。
 独学で全工程を"正しく"回せる仕組みを持っている人が、
 そもそも存在しないということです。」
```

**穴の書き出しシート（オプチャに配布）**:
```
【一人では埋められない穴リスト】

□ 商品設計（何を作ればいいか迷う）
□ LP/セールスコピー（文章が書けない・書いても自信がない）
□ 集客（フォロワーはいるが見込み客への転換方法が分からない）
□ 教育コンテンツ設計（Day1〜5をどう設計すれば売れるか分からない）
□ AIの使い方（プロンプトが分からない・エラーが出たときに対処できない）
□ クロージング（面談やDMで成約させる方法が分からない）
□ フィードバック（これで合ってるか誰に聞けばいいか分からない）
□ 継続力（一人だと途中で止まる・孤独感）
□ その他: ____________

→ 書き出した穴の数: ___個
```

**設計意図**:
- 「環境が必要」を主催者が言うのではなく、参加者が自分の言葉で書く
- 書いた瞬間に認知的不協和が発動（「書いたのに行動しないのはおかしい」）
- 第3幕で「さっき書いた穴、全部埋まります」と照合することでセールスが「答え合わせ」になる

---

### ワーク⑥ 課題解決後のビジョンを書く（新規追加・ライトワーク）

> Day4第2幕の後半、穴の書き出しの直後に追加。ワーク⑤と合わせて流れを作る。

**目的**: 課題解決後のイメージの明確化 → セールスへの自然な接続

**強める要素**: 課題解決後のイメージの明確化

**タイプ**: 問いかけ型

**所要時間**: 2〜3分

**ファシリテーション**:
```
「もう一個だけ。

 さっき書いた穴、全部埋まったとしたら。
 6ヶ月後、どんな状態になってる？

 一文で書いてみてください。」

（1分）

「書いた人、チャットに流してもらっていいですか？
 その状態、作りに行きましょう。」
```

**設計意図**:
- 「穴がある（課題）」→「穴が埋まった未来（解決後）」の対比で欲求を最大化
- この一文がセールストークの中で「あなたがさっき書いた状態、一緒に作りにいきましょう」と繋がる
