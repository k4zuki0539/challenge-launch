## 1. 準備項目 A〜D

サブコピー: 1000万ローンチを再現するAI×Xの設計図

### B. メインターゲット像

## 2. 問題提起設計

### 問題提起3: 本質的な壁（★Day4で扱う・最重要・梯子落とし）

**壁**: 「ローンチは"設計図"だけでは完走できない。一人で全工程を回しきることの構造的限界」

> **なぜDay4か**: Day3で「できそう！」と高揚させた直後に落とす。高いところから落とすほど衝撃が大きく、そのまま解決策（=わどAIスクール）への導線が自然になる。前回はDay3に梯子落とし→Day4が薄くなり面談遷移が悪化した反省を踏まえた配置。

| 項目 | 内容 |
|------|------|
| **壁の本質** | 商品設計→LP→集客→教育→セールスの全工程を、独学・独力で「正しく」回せる人はほぼいない。どこかで必ず詰まり、フィードバックがないまま間違った方向に進み続ける |
| **なぜ独学では無理か** | ①自分の商品・LPを客観視できない ②集客と販売を同時並行で回す実行力の限界 ③「これで合ってるのか？」という不安が行動を止める ④孤独で継続できない（前回CL受講者の最大の挫折要因） |
| **Day3との落差設計** | Day3で「設計図も分かった、AIも使える、自分にもできそう！」→ Day4冒頭で「…本当に？一人で全部回せる？」と現実を突きつける |
| **究極の解決策** | 再現性あるカリキュラム＋プロの伴走＋同じ志を持つ仲間の環境 |
| **感情の着地** | 「設計図は分かった。でも一人じゃ完走できない。環境が必要だ」→ そのままセールスへ接続 |

#### ★Day4 三幕構成（AI検証で追加）

> **問題**: 梯子落とし→即セールスだと「落として売る」パターンがバレる。59万スクール経験者は「体験→落とす→売る」を知っている。
> **対策**: 梯子落としとセールスの間に「自分で気づく間」を挟む。セールスが「売り込み」ではなく「答え合わせ」になる。

```
【第1幕】梯子落とし（25分）
  全工程を提示 → 「これ、一人で全部回せる？」
  メインメッセージ: 一人で全工程を回すのは構造的に無理がある
  + ユースケースの1つとして: 過去に独学やスクールで挫折した経験がある人には
    「あれは能力の問題じゃなく構造が足りなかっただけ」と軽く触れる（全員向けではない）

【第2幕】自分で気づくワーク（10分）★ここが肝
  「あなたのローンチ計画、一人で埋められない穴はどこ？書き出してみてください」
  → 参加者が自分の言葉で「環境が足りない」と気づく
  → この気づきがあるから、第3幕のセールスが「答え合わせ」になる
  ※ 主催者が「環境が必要」と言うのではなく、参加者が自分で辿り着く設計

【第3幕】解決策提示 → わどAIスクール → 面談案内（40分）
  「さっき書き出した"穴"、それを全部埋めるのがこの環境です」
  → 第2幕のワーク結果と照合する形でスクールの要素を提示
  → 面談は「個別面談」ではなく「あなた専用ローンチ設計セッション」としてリフレーム
  + QR提示前に面談で得られるものを明示:
    ①あなた専用のローンチロードマップ ②最初の30日のアクションプラン ③合わなければ正直にそう言う
```

**チェック:**
- [x] 不可逆性: 独学の限界は構造的であり、気合いでは解決しない
- [x] 金額正当性: 「一人で何年も遠回りする vs 環境に投資して最短で結果を出す」
- [x] 梯子落とし→セールスの流れが1Day内で完結: Day4内で「壁の直面→自分で気づく→解決策=商品」
- [x] **操作感の排除**: 第2幕で「自分で気づかせる」ことで、セールスが売り込みではなく答え合わせになる
- [x] **過去の失敗への配慮**: 第1幕でスクール失敗経験者にユースケースの1つとして触れる（メインメッセージではない）

### 問題提起2: 方法論の壁（Day2で扱う）

**壁**: 「"発信すれば売れる"は幻想。売れる人は"ローンチの設計図"を持っている」

| 項目 | 内容 |
|------|------|
| **壁の本質** | ほとんどの人が「とりあえず発信」「とりあえずコンテンツ作成」をしている。しかし売れる人と売れない人の差は「設計図（ローンチ構造）を持っているかどうか」 |
| **ありがちな間違い** | ①商品を作ってから集客する（順番が逆） ②フォロワー＝見込み客だと思っている ③AIを「コンテンツ量産ツール」としか使っていない ④「良い商品を作れば売れる」と信じている |
| **解決策（Day2で提示する価値）** | **オートローンチシステム**の全体構造を公開。「集客→教育→販売」の設計図と、AIで各工程を高速化する具体的な方法を1つ見せる |
| **感情の着地** | 「そういうことか！この設計図なら自分にもできそう…でも全部一人でやれるか？」 |

**チェック:**
- [x] 知る前と後で行動が変わるか: 「とりあえず発信」→「設計図ベースで動く」に変わる
- [x] 限界の示唆: 「設計図は分かった。でも自分のケースに当てはめると？」→Day3の実践へ
- [x] 優越感: 「周りがまだ闇雲にやってる中、自分は構造を知った」

### 問題提起1: 表面的な壁（Day1で扱う）

**壁**: 「"自分には売れるものがない"という思い込み」

**★Day1冒頭: 不信感ケアパート（AI検証で追加）**
> Day1の問題提起に入る前に、**「過去の失敗に正面から触れるパート」を必ず入れる**。
> ターゲットの多くは高額スクールで失敗した経験があり、「また同じか」という防御反応が最初に来る。
> これをケアしないと、どんなに良い内容でも耳に入らない。
>
> **設計意図**: 逆説的介入（パラドキシカル・インターベンション）
> 「疑ってOK」と先に言うことで、逆に警戒が下がるメカニズムを使う。
>
> **トークの方向性**:
> 1. 「今日、疑いながら聞いてる人いると思います。正直、それが正しい」
> 2. 「過去にスクールやコンサルで大金払って成果ゼロだった人もいるはず。その警戒心は"学び"です」
> 3. 「だから今日は"信じてください"とは言いません。ただ、5日間の中で"自分で判断する材料"を渡します」
> 4. 「信じるかどうかは5日後のあなたが決めればいい」

| 項目 | 内容 |
|------|------|
| **壁の本質** | 「商品を持っていない」のではなく「自分のスキル・経験が商品になると気づいていない」。既にX運用してフォロワーもいるのに、「自分なんかが…」と過小評価している |
| **常識破壊** | 「商品がない」のではなく「商品の作り方を知らないだけ」。あなたが当たり前にできることは、できない人から見れば喉から手が出るほど欲しいスキル |
| **具体的な気づき** | ①あなたのスキル×フォロワーは既に「資産」 ②AI時代は「作業者」から「設計者」に回った人が勝つ ③1000万ローンチは特別な才能ではなく「再現可能な構造」で生まれる |
| **感情の着地** | 「え、自分のスキルが商品になるの？…でも具体的にどうやって？」 |

**チェック:**
- [x] 痛みに寄り添えているか: 「売れるものがない」は最大のペイン
- [x] シンプルか: 「思い込みを外す」というワンメッセージ
- [x] Day2への期待感: 「じゃあ具体的にどうやるの？」→方法論の壁へ
- [x] **不信感ケア**: 冒頭で「疑ってOK」と正当化 → 防御反応を解除してから本題に入る

通勤電車でXを眺めていたら、タイムラインに「The Launch Code 5Days」の告知が流れてきた。
「オートローンチ…？」。聞いたことない言葉に指が止まる。
LPを開く。「ローンチの全工程をAIでほぼ自動化する仕組み」——普通のAI副業や情報商材とは明らかに違う匂いがする。

一度は閉じる。でも帰宅後、子どもを寝かしつけた23時。リビングのソファでスマホを開き直す。
59万溶かした記憶がチラつく。でも「無料だし、"オートローンチ"の正体だけ知りたい」と自分に言い訳しながら登録ボタンを押す。

**心の中**: 「どうせまた煽りだろ。でも…"オートローンチ"って何だ？仕組みだけでも知りたい」

### 【Day1】問題提起1 — 「自分には売れるものがない」の破壊

#### セミナー冒頭 — 不信感ケアパート（最初の5〜10分）

> **台本指針**: いきなり本題に入らない。最初に「聞く姿勢」を作る。
> 参加者の多くは警戒心を持って入ってくる（情報商材への不信感、過去の失敗経験など）。
> ここでやるのは「売り込み」ではなく「防御反応の解除」。

トークの流れ:
1. 「今日、"また怪しいやつか"と思いながら見てる人、いると思います。正直、それが正しい」
2. 「ネット上には煽りや誇大広告が溢れてる。疑って当然です」
3. 「だから今日は"信じてください"とは言いません。5日間で"自分で判断する材料"を渡すだけです」
4. 「信じるかどうかは、5日後のあなたが決めてください」

**参加者の反応**: 「いつもと入り方が違うな」→ 「攻撃的な疑い」が「様子見の好奇心」に変わる。
チャット欄が動く。「疑ってます笑」「それ言ってくれるの助かる」。空気が少し緩む。

#### セミナー中の情景 — 問題提起パート

不信感ケアで壁が少し下がった状態で、本題に入る。
「あなたが"当たり前"にできること、それを喉から手が出るほど欲しい人がいる」

参加者は自分のスキルを思い浮かべる。映像制作、ライティング、デザイン、エンジニアリング…
「商品を持っていない」のではなく「自分のスキルが商品になると気づいていないだけ」。
この気づきが入る。ただし、まだ完全には信じていない。
「面白い考え方だけど、本当に売れるのかな？」が正直な反応。
でも冒頭で「疑ってOK」と言われているから、「まあ疑いながら聞き続けてもいいか」と思える。

#### ワーク: スキル棚卸し
セミナー後、「自分が当たり前にできること」を書き出す。
意外と出てくる。5個、10個…「こんなにあるのか」。
オプチャに投稿すると、同じようにスキルを持ちながら売れてない人がたくさんいる。
「自分だけじゃない」→ 共感と安心。

**Day1の感情着地**: 「まだ信じてはいない。でも"疑いながら聞いていい"って言われたから、もう少し見てみるか。明日の設計図ってやつ、気になるし」
（半信半疑 6 : かすかな希望 3 : 好奇心 1 — まだ希望は上回っていない）

### 【Day2】問題提起2 — 「闇雲に発信しても売れない」の解決

#### セミナー中の情景
今日はPCで参加。画面にオートローンチシステムの全体図が映し出される。
「集客→教育→販売」という構造を見て、タケシは固まる。
自分がやってきたことを振り返る。Xで毎日「映像Tips」を投稿して、たまにnoteでテクニック記事を書く。
フォロワーは800人。でも収益はゼロ。
「…俺、"教育"も"販売"も何もやってなかった。ただ発信してただけだ」
さらに「AIでLP文章もセールス導線も半自動で作れる」というデモを見て、目が釘付けになる。
「これ…59万のスクールでは一切教わらなかったぞ」

#### ワーク提出の情景
オートローンチシステムの各ステップに、自分のスキル（映像制作講座）を当てはめてみる。
「集客: Xで映像制作の悩みに刺さる投稿→教育: 5日間の無料講座→販売: 個別サポートプラン…」
テンプレートに沿って埋めていくと、ぼんやりしていた「自分のローンチ」が初めて形になる。
オプチャに提出すると、他の参加者から「映像制作の講座、あったら受けたいです！」とコメントが付く。

#### Day2の感情着地
頭の中でオートローンチシステムの全体図がグルグル回っている。
自分のスキルを当てはめたワーク結果を見返す。「理屈は分かった。でも実際にできるのか？」
**「分かる」と「できる」は別の話**。この壁はDay3の実践体験で越えさせる。

**Day2の感情着地**: 「構造は理解できた。今まで知らなかった全体図が見えた。でも"分かる"と"できる"は別。明日のワークショップで実際にやってみないと何とも言えない」
（理解・納得 6 : 「自分にできるか？」の疑問 5 : 不安 4 — 前向きだがまだ慎重）

### 【Day3】実践・体験 — 「できた！」の高揚 ★感情のピーク

#### セミナー中の情景
今日はワークショップ型。PCの前で、実際にAIツールを触りながらローンチの一部を作ってみる。
「じゃあ今から、あなたのスキルを使って"商品コンセプト"をAIで作ってみましょう」
プロンプトをコピペして、自分の映像制作スキルの情報を入力。
30秒後、画面に「映像クリエイターのための"売れる自分商品"設計講座」というコンセプト案と、ターゲット像と、LP構成案が出てくる。
「…え、これ30秒でできるの？」
隣の席（オプチャ）では、イラストレーターの人が自分の商品コンセプトを晒して「私にもできた！」と興奮している。
ライターの人は「記事代行じゃなくてライティング講座を商品にすればいいのか」と書き込んでいる。
チャット欄が一気に活気づく。

#### ワーク提出の情景
セミナー中に作った商品コンセプトを、さらにブラッシュアップして提出。
AIでLP文章のドラフトまで出してみる。「これ…普通にそれっぽいLPになってる」
ただ、冷静な部分も残っている。「でもこれAIが書いたテンプレでしょ？実際に売れるかは別の話だよな」
オプチャに投稿。すると、数人から「映像制作の講座、これいくらなら受けたいですか？」という問いに
「5,000円なら即買いです」「1万でも全然アリ」とコメントが付く。
これは刺さった。AIの出力じゃなくて、**自分のスキルに値段がついた瞬間**。
「…え、俺のスキルに金額出してくれる人、いるのか」

#### Day3終了後の行動
セミナーが終わった後、PCの前でしばらくオプチャのコメントを読み返している。
「5,000円なら即買い」——この言葉が頭から離れない。
AIでもう少しLP文章を触ってみる。「…これ、もうちょっと直したらマジでいけるんじゃないか？」
ただし、59万の経験があるから「簡単にできた＝罠かも」という疑いも同時に走ってる。
「こんな簡単にできるなら、なんでみんなやってないんだ？…何か落とし穴があるのか？」
妻には今日も話さない。でも、PCを閉じた後も頭の中がざわざわしている。

セミナーの最後に言われた一言が引っかかっている。
**「今日やったのはローンチの"最初の1ステップ"。全体像はこちら（全工程図を一瞬見せる）。…次回、大事な話があります」**
「…大事な話って何だ？」

**心の中**: 「正直、驚いた。自分のスキルに値段をつけてくれる人がいる。AIでここまで形にできる。…でも全工程を見た時のあの量。本当にこれ全部できるのか？次回の"大事な話"って何だろう」
（驚き・手応え 7 : 「本当に使えるのか？」の懐疑 3 : Day4への好奇心 — ピークだが天井ではない。オープンループが残っている）

### 【Day4】梯子落とし → セールス ★最重要Day（三幕構成）

#### 第1幕: 梯子落とし（25分）

> **台本指針**: Day3の「手応え」を認めつつ、全工程の現実を突きつける。
> メインメッセージは「一人で全工程を回すのは構造的に無理」。
> スクール失敗者への救済はユースケースの1つとして軽く触れる程度。

セミナーが始まる。Day3の余韻がまだ残っている。
「Day3で皆さん、商品コンセプトやLPドラフトを作りましたよね。すごかった。
でも今日は"大事な話"をすると予告しました。正直に話します」

画面にローンチの全工程が表示される。
商品設計 → LP制作 → 集客（X運用・広告） → リスト獲得 → 教育コンテンツ → セールス → 個別対応 → アフターフォロー
「これ、全部を"同時に"回す必要があります」

参加者の空気が変わる。Day3で作ったのはコンセプトとLPの"ドラフト"。ほんの一部だった。
集客は？リスト獲得の導線は？教育コンテンツの中身は？セールスのシナリオは？反応が悪い時の軌道修正は？
Day3の終わりに感じた「落とし穴」の予感が、ここでハッキリ形になる。

「あなたのLP、誰がレビューしてくれますか？」
「集客が止まった時、誰に相談しますか？」
「セールス文が刺さらない時、何を直せばいいか分かりますか？」

ここで、**過去に独学やスクールで挫折した経験のある参加者**にも一言触れる（全体の中のユースケースの1つとして）:
「過去に一人でやってみて、途中で止まっちゃった経験がある人もいると思います。
 あれは能力の問題じゃなくて、**構造が足りなかっただけ**です」
——これはメインメッセージではなく、該当者の自己嫌悪を軽くする一言。全員に刺さる必要はない。

メインの着地は全参加者共通: **「設計図は分かった。AIも使える。でも全工程を一人で回しきるのは、構造的に無理がある」**

**第1幕の感情着地**: Day3の手応え → 全工程の現実 →「じゃあ自分に足りないのは何だ？」を考え始める

#### 第2幕: 自分で気づくワーク（10分）★ここが肝

「ここで1つワークをします。今からあなたのローンチ計画を見てください。
 Day2で作った設計図、Day3で作ったコンセプトとLP。
 この中で、**"一人では埋められない穴"はどこですか？** 3つ書き出してください」

タケシはDay3のワーク結果を見返す。コンセプトはある。LPドラフトもある。
でも…
- 「このLPで本当に集客できるか、誰にもレビューしてもらえない」
- 「集客とセールスを同時に回す方法が分からない。本業もある」
- 「途中で挫折した時、一人で立て直す自信がない。前も同じだった」

書き出してみると、全部「一人であること」が原因だと気づく。
オプチャにワーク結果が流れてくる。他の参加者も同じようなことを書いている。
「レビューしてくれる人がいない」「孤独で続けられる自信がない」「軌道修正ができない」

**誰も「環境が必要」とは言われていない。でも全員が自分の言葉で同じ結論に辿り着いている。**

**心の中**: 「…全部、一人であることが問題なんだ。設計図はある。AIもある。スキルもある。でも"一人"じゃ無理だ。これは能力の問題じゃない。構造の問題だ。さっき言われた通りだ」
（自分で気づいた納得感 — 「売り込まれた」のではなく「自分で辿り着いた」実感）

#### 第3幕: 解決策提示 → わどAIスクール → 面談案内（40分）

> **台本指針**: 第2幕で参加者が書き出した「穴」に対する答え合わせとして商品を提示する。
> 「売り込み」ではなく「自分が見つけた課題への解決策」というフレーム。

「さっき書き出した"穴"。その穴を全部埋めるのが、これから紹介する環境です」

参加者は身構える。でもいきなり「売り込まれた」感が薄い。
**自分でワークに書き出した「穴」に対して、1つずつ解決策が提示される**からだ。
- 「LPのレビューがない → プロによる個別フィードバック」
- 「集客とセールスの同時進行が無理 → カリキュラムで段階的に実行」
- 「一人では続かない → 同じ志を持つ仲間のコミュニティ」

スクール生の事例を複数紹介。
**様々なバックグラウンドの人が結果を出している**ことを見せる:
- 独学でやって途中で止まった人 → 環境に入って3ヶ月で初ローンチ
- スキルはあったが商品化できなかった人 → AIで商品設計、○ヶ月で売上○万円
- 別のスクールで失敗した経験がある人 → 「設計図＋環境」で今度はうまくいった

面談の案内。ただし「個別面談」ではなく、**「あなた専用ローンチ設計セッション（30分）」**。
- 「Day3で作った商品コンセプトを持ってきてください」
- 「あなたのスキル×ターゲットに合わせた、最初の30日のアクションプランを一緒に作ります」
- 「セッション後、スクールが合わないと判断すれば正直にそう言います」

**参加者の心理**: 「面談」ではなく「設計セッション」。Day3で自分が作ったコンセプトを持っていける。
「売り込まれる場」ではなく「自分のローンチ計画を一緒に作ってもらえる場」。
合わなかったらやめればいい、と思えるハードルの低さ。

**Day4の感情着地**: 「自分で穴を書き出して、その穴に対する答えが出てきた。まずは設計セッションで自分のケースを見てもらおう。合わなかったらやめればいい」
（自分で辿り着いた納得 → 「試してみよう」という静かな決断 — 熱狂ではなく論理的な判断）

---

### 【Day5】総括＋最終セールス

#### セミナーの情景
Day1からの旅路が1枚のスライドにまとめられている。

「Day1: "自分には売れるものがない"と思っていた → スキル×フォロワーが資産だと気づいた」
「Day2: 闇雲に発信していた → "設計図"という概念を手に入れた」
「Day3: AIで実際に商品を作ってみた → "自分にもできる"と体感した」
「Day4: 全工程を一人で回す限界に気づいた → 環境の必要性を理解した」

タケシは5日前の自分を思い出す。通勤電車でスワイプして飛ばそうとした自分。
「あの時登録してなかったら、今もXで毎日Tips投稿してるだけだったな」

セールスの締め切りが告知される。
まだ面談を予約していない参加者に向けて、最後のメッセージが投げかけられる。

「5日間で設計図を手に入れた。AIで商品が作れることも体験した。
 でもここで"面白かった"で終わったら、また明日から同じ毎日が始まる。
 来月も同じXのタイムラインを眺めて、同じ焦りを感じる。
 1年後も"いつかやろう"と思っている。
 環境に入るかどうかは、今日決めなくていい。
 でも"話を聞いてみる"かどうかは、今日決められる」

#### Day5終了後の行動
面談予約済みのタケシは、面談日までにDay3で作った商品コンセプトをもう一度見返す。
「これを持って面談に行こう。具体的に何から始めればいいか聞いてみよう」
Xに「5日間で人生の設計図が変わった。行動する」とだけポストする。

面談未予約の参加者の一部は、この日の夜に予約を入れる。
「やっぱり、このまま何もしないのは嫌だ」

**心の中**: 「5日前の自分と今の自分、見えてる景色が全然違う。設計図がある、AIがある、スキルがある。あとは環境だけ。ここで動かなかったら、また同じ1年だ」
（覚悟 + 静かな決断）

## 6. Day3 具体設計 — AIで何を触らせるか・相互値付けの仕掛け

### 体験の設計原則
- **完走ではなく「最初の1ステップ」の体験** に絞る（完走させようとするとハードルが上がる）
- 30秒〜1分で「形になる」体験を最優先
- 「AIが書いたテンプレ」ではなく「自分のスキルに合わせた出力」と感じさせる

---

### ワーク設計: 3ステップ体験（合計20〜25分）

#### ■ Step 1: 商品コンセプト生成（8分）

**目的**: 「自分のスキルが商品になる」を30秒で体感させる

**ワーク手順:**
1. オプチャに以下のプロンプトを貼る（参加者はコピペするだけ）
2. 自分のスキル情報を3つ入力してAIに投げる
3. 出てきたコンセプト案をチャット欄に1行投稿

**プロンプト（オプチャに配布）:**
```
以下の情報をもとに、私が作れる「スキルを商品化した講座のコンセプト」を提案してください。

【私のスキル】○○○
【得意なこと・できること】○○○
【これまでの経験・実績】○○○

出力形式:
・商品タイトル（キャッチーに）
・ターゲット（どんな人に向けか）
・この商品で得られる変化（ビフォー→アフター）
・LP1行キャッチコピー

簡潔に、でもビジネスで使えるレベルで。
```

**ファシリテーションポイント:**
- 「入れたら30秒で出てきます。驚かないでください」と前置き
- 出てきたらチャット欄を促す（「1行だけでいい、タイトルだけでも」）
- リアクションが盛り上がったら「これ、今日だけで商品の骨格ができた」と強調

---

#### ■ Step 2: LP文章のドラフト生成（10分）

**目的**: 「LP文章まで形になる」驚きを与え、高揚感を強化する

**ワーク手順:**
1. Step 1で出たコンセプトをそのまま使う
2. 以下のプロンプトを使ってLP冒頭文を出力
3. 出力結果をオプチャに「値付けチャレンジ投稿」として投稿（→相互値付けへ接続）

**プロンプト（オプチャに配布）:**
```
以下の商品コンセプトをもとに、LP（ランディングページ）の冒頭文を書いてください。

【商品タイトル】
【ターゲット】
【得られる変化（ビフォー→アフター）】

出力形式:
・ファーストビューのキャッチコピー（15文字以内）
・サブキャッチ（30〜40文字）
・LP冒頭の共感文（ターゲットの悩みに刺さる3〜4行）

口語調で、ターゲットが「これ私のことだ」と思う表現で。
```

**ファシリテーションポイント:**
- 「AIが出した文章をそのまま使う必要はない。でも、方向性はこれで合ってる」と伝える
- 「自分で一から書こうとしたら何時間かかる？」という問いかけで価値を実感させる
- 「完成ではなく"たたき台"が30分でできた」ことを強調

---

#### ■ Step 3: 全工程図の一瞬提示 → オープンループ（5分）

**目的**: 「全体はこれだけある」を見せてDay4への好奇心を作る

**ファシリテーション:**
1. ローンチ全工程図をスライドで1枚表示（詳細は見えなくていい）
2. 「今日やったのは、ここ（1ステップ目を指す）だけです」
3. 「これを全部一人でやるとしたら…どう思いますか？」（問いかけで終わる）
4. 「大事な話は次回します。必ず来てください」

**スライド案（1枚）:**
- ローンチ全工程: 商品設計 → LP作成 → 集客設計 → 教育コンテンツ → セールス → クロージング → フォローアップ
- 今日のパートを赤丸で囲む（商品設計の一部のみ）
- 残りの工程はグレーアウト（見えるけど詳細は読めない状態）

---

### 相互値付けの仕掛け

> AIが出した出力に感動させるだけでは「AI面白い」で終わる。
> **自分のスキルに他者が価値を認める**体験が「自分でもやれる」に変わる。

#### 「値付けチャレンジ」の仕組み

**オプチャ投稿フォーマット（セミナー中に案内）:**
```
【値付けチャレンジ】

商品名: ○○○
対象: ○○に悩む○○の人
得られる変化: ○○→○○
AIで出たLP冒頭キャッチ: 「○○○」

この講座、いくらなら即決で買う？コメントで教えてください！
```

**主催者側のオペレーション:**
1. セミナー中に「今から投稿時間3分取ります。値付けチャレンジ投稿してください」と宣言
2. 最初に投稿が来たら主催者がリアクション + 値段コメントを入れる（旗手として機能）
3. セミナー後も「スタッフが全員の投稿にコメントします」と予告してアクション設計
4. 翌日（Day4前）のオプチャリマインドで「昨日の値付けチャレンジ、○○さんに○万円の声がついていました」と紹介

| 何が起きるか | 参加者の感情 |
|-------------|------------|
| 見知らぬ人が「○円なら即買い」とコメント | 「俺のスキルに価値がある」という確信 |
| 自分も他の人にコメントする | 「この人も買えるじゃん」→ 自分への転写 |
| 複数人から値段がつく | 「市場価値が実証された」感覚 |

**注意点:**
- 「高い値段をつけてください」と誘導しない（不自然に感じる）
- 0円・安すぎるコメントが来ても否定しない。「なぜそう思ったか」を掘り下げることで逆に教育になる
