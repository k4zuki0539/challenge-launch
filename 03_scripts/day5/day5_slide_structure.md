# Day5 スライド構造設計書

> Day1〜4の構造を継承しつつ、Day5特有の「総まとめ + 最終セールス」設計を反映。
> **今日は感謝と成長の確認が主役。スライドは旅路の記録として機能させる。**

---

## ■ スライド全体の流れ（ブロック構成）

```
[BLOCK A] コンプリート特典の提示（★最初に）
[BLOCK B] アウトプット紹介（5日間通じて + 商品代弁）
[BLOCK C] 成長の可視化（5日前vs今）
[BLOCK D] 問題提起1〜3の総ざらい（一気通貫）
[BLOCK E] 理想の未来 + 状態変化の認識（セールス橋渡し）
[BLOCK F] セールスパート
[BLOCK G] クロージング + コンプリート特典受け取り案内
[BLOCK H] 付録（Q&A）
```

### [BLOCK C] 成長の可視化（4〜5枚）★

#### C-1: 5日前を思い出させる
- 型: チャット質問スライド

```

「また怪しいやつかな」と思いながら登録した方
「最後まで来るとは思わなかった」という方

チャットに正直に送ってください
```

#### C-2: ★5日間の変化一覧表（メインスライド）
- 型: 4行の比較表（Day別・Before/After）

| Day | 出発点（5日前） | 今のあなた |
|-----|--------------|----------|
| Day1 | "売れるものがない"という思い込み | スキル×リストが資産だと確信 |
| Day2 | 発信しても売れない理由が分からない | 設計図（集客→教育→販売）が頭に入った |
| Day3 | "分かる"と"できる"の間 | AIで商品を作り、需要をリアルに確認した |
| Day4 | 一人でやるのが当たり前だと思っていた | 環境の必要性を、自分の言葉で理解した |

> **デザインポイント**: 左列（出発点）をグレー/暗め、右列（今）を明るい色で対比させる

#### C-3: 「あなたが変えた」スライド
- 型: 強調大テキスト

```

```

### [BLOCK D] 問題提起1〜3の総ざらい（4〜5枚）

> ★5日間の教育を一気通貫で整理する。コンパクトに、フロー図で見せる。

#### D-1: 出発点の確認
- 型: 参加前の状態リスト

```
5日前のあなたがいた場所

・Xで毎日発信しているのに収益がゼロ
・スキルはあるけど"自分の商品"がない
・AIは使ってるけど稼ぎに直結していない
・"自分なんかが売れるのか"という不安
```

#### D-2: 問題提起1〜3の総ざらいフロー（★凝縮スライド）
- 型: 縦フロー図（7ステップ）

```
① あなたがいた現状
  ↓
② 理想の未来（スキルで収益化できている状態）
  ↓
③ 壁1: "売れるものがない"という思い込み
  → 壁の正体: 見つけ方を知らなかっただけ
  ↓
④ 壁2: 発信しても売れない
  → 壁の正体: 設計図（ローンチ構造）がなかっただけ
  ↓
⑤ 壁3（最大の壁）: 一人で全工程は完走できない
  → 壁の正体: 環境がなかっただけ
  ↓
⑥ 3つの壁の共通点
  「どれも、あなたの能力の問題ではなかった」
  ↓
⑦ 3つの壁を越えるための唯一の答え
  → 設計図 × 環境
```

#### D-3: 「あなたの能力の問題ではなかった」スライド
- 型: 強調大テキスト


## ■ Day5特有の設計ポイント

### 1. コンプリート特典は最初の1分で見せる
来てよかったという満足感を最初に作る。
特典が具体的であるほど「使えそう」という期待が高まる。

### 2. 5日間の変化一覧表はデザインに力を入れる
Day5唯一の「まとめ系スライド」として、デザインに一番時間をかけていい。
Before/Afterの対比が視覚的に伝わることが重要。

### 3. セールスは「繰り返し」ではなく「最後だから正直に」スタンスで
Day4と同じ内容でも、語り口を変える。
「5日間を共にしたからこそ伝えたい」という感情を乗せる。

### 4. QR提示後は焦らない
申し込みを検討している人が動く時間を作る。
沈黙は「時間を与えている」という演出として使う。
Q&Aの時間もQRを消さない。

### 5. 「Day4を見ていない人がいる」前提で商品案内をする
同じスライドを使っていいが、「今日初めて聞く方のために改めて」という一言を入れる。

---

## ■ スライド制作上の注意点

- **5日間の変化一覧表** — デザインに最も力を入れる
- **QRスライドはシンプルに** — 余計な情報は入れない
- **特典スライドは具体的に** — テンプレートの名前を書く
- **最後の一言スライドは文字少なめ** — 感情が乗るシンプルな設計
