## ■ スライド全体の流れ（ブロック構成）

```
[BLOCK A] アウトプット紹介（Day2課題の成果紹介）
[BLOCK B] ロードマップ + 振り返り（厚め）+ Today's Goal
[BLOCK C] 解説（設計図を自分に当てはめる思考）
[BLOCK D] AIワーク① 商品コンセプト生成
[BLOCK E] AIワーク② LP文章ドラフト生成
[BLOCK F] 相互値付けチャレンジ
[BLOCK G] オープンループ（★Day3のクライマックス）
[BLOCK H] クロージング + Day4告知
[BLOCK I] 付録（Q&A）
```

> ★Day3は「体験」が主役。スライドは体験を誘導するガイドに徹する。
> ★権威性・自己紹介はDay1で完了しているため不要。

### [BLOCK A] アウトプット紹介（4〜5枚）

#### A-1: 冒頭チャット確認
- 「昨日の課題、提出してくれてありがとう！みんなの3要素、全部確認した」
- 「今日のワークで使うから手元に出しておいて。PC準備できてる方は「準備OK」と送ってください」
- 型: 提出前提の感謝 + 準備促進（「書いてみてどうだった？」と感想を促してもOK）
- ★「書けた方いますか？」は使わない（提出が参加の前提のため）

#### A-2〜A-4: アウトプット紹介スライド（3〜4件）
- 型: 参加者投稿の3要素 + コメント

```
スキル: ___________
① 誰: ___________
② 現状: ___________
③ 理想: ___________

→ なぜこれが商品になるか（1〜2行コメント）
```

> **ポイント**: 「今日のワークでさらに深まる」という予告として紹介する

#### A-5: ブリッジスライド
```
昨日: 商品の「芯」ができた
今日: AIで30秒で"形"にする
```

| 返信文を自動化する | セールスフローを設計する |

## ■ Day3特有の設計ポイント

### 1. スライドは「体験のガイド」に徹する
Day1・Day2はスライドがメイン。Day3はAIワークとチャット交流がメイン。
スライドは「今何をすべきか」を示すガイド役に絞る。

### 2. プロンプトスライドは可読性最優先
コピペしやすい大きさで表示する。
モバイル参加者への配慮として、プロンプトはLINE/オプチャにも事前配布しておく。

### 3. 全工程図はあえて「圧倒感」を演出する
全ステップを一枚に詰め込んで表示する。
「こんなにある」という圧倒感がDay4への効果的なオープンループになる。
5〜10秒だけ表示してすぐ閉じる演出を必ず入れる。

### 4. 相互値付けの盛り上げを配信側でサポートする
配信担当がオプチャをリアルタイムモニタリングし、コメントが付いた投稿をピックアップして読み上げる。
「○円なら即買い」コメントをスクリーンに映すとさらに効果的。

### 5. Day4への「謎感」を意図的に残す
ロードマップのDay4を「🔒」表示にする。
クロージングで「大事な話がある」と2〜3回繰り返す。
内容を絶対に言わない（ここが肝）。

---

## ■ スライド制作上の注意点

- **AIプロンプトスライドは大きく** — コピペが最優先
- **ワーク中スライドはシンプルに** — 「やってみてください」だけでいい
- **全工程図は情報量多め** — 圧倒感を出すためにあえて詰め込む
- **Day4の告知スライドは「謎」を残す** — 内容は書かない
