# Day2 スライド構造設計書

> Day1スライド構造を基礎に、Day2特有の要素を差し込んだ設計書。
> **内容は今回のコンセプト（The Launch Code 5Days）に合わせて設計。**
> **構造・インタラクションパターンはDay1から継承。**

---

## ■ スライド全体の流れ（ブロック構成）

```
[BLOCK A] アウトプット紹介（Day2特有）
[BLOCK B] 企画目的 + ロードマップ + Today's Goal
[BLOCK C] Day1の振り返り（Day2特有）
[BLOCK D] メインコンテンツ1（問題提起1の解決策: 商品化3要素）
[BLOCK E] メインコンテンツ2（問題提起2: 設計図がない人の末路）
[BLOCK F] ワーク
[BLOCK G] クロージング + Day3告知
[BLOCK H] 付録（Q&A）
```

> ★Day2は権威性・自己紹介ブロックは不要（Day1で済んでいる）
> ★代わりに「アウトプット紹介」と「Day1振り返り」が追加される

### [BLOCK A] アウトプット紹介（4〜6枚）★Day2特有

#### スライド型: "Day1ワーク紹介 + 承認"

**A-1: 冒頭チャット確認スライド**
- 「昨日の課題、提出してくれてありがとう！みんなのアウトプット全部見た。早速シェアしていくね」
- 型: 提出前提の感謝 + アウトプット共有への導入（「やった感想」をチャットに促してもOK）
- ★「やってみた方いますか？」は使わない（提出が参加の前提のため）

**A-2〜A-4: アウトプット紹介スライド（3件程度）**
- 型: 参加者投稿のスクショ + コメント
- 構造: 
  ```
  ○○さんの投稿: 「〜〜〜」
  ↓
  コメント: なぜこれが商品になるか（1〜2行）
  ```
- ポイント: 「本人には当たり前でも、他者には価値がある」を毎回伝える

**A-5: まとめスライド**
- 「みんな、すでに資産を持っている」
- 型: 一行メッセージ（大テキスト）

> **インタラクション**: アウトプット未提出者へは「今日の内容聞いてから書いてみてください」と逃げ道を作る

### [BLOCK D] メインコンテンツ1 — 商品化の3要素（6〜8枚）

> ★問題提起1の解決策（具体アクション）パート

### [BLOCK E] メインコンテンツ2 — 設計図の正体（10〜12枚）

> ★今日の最重要パート。問題提起2「方法論の壁」を扱う。

#### E-1: 共感質問スライド
- 型: チャット質問

```
「商品を出したのに売れなかった」
「"興味あります"と言われたのに買ってもらえなかった」
「毎日発信してるのに収益につながらない」

チャットに「ある」と送ってください
```

#### E-2: 常識破壊スライド（★このDayの核心）
- 型: 強調テキスト2段構成

```
（小さく）売れない理由は、

商品が悪いわけでも
発信量が足りないわけでも
フォロワーが少ないわけでもない。

（大きく）
「設計図（ローンチ構造）を
  持っているかどうか」
         の差だけだ。
```

#### E-3〜E-6: 4つの間違いパターン（1パターン1枚）
- 型: パターンラベル + 一行説明 + 正解の方向性

**パターン①: 商品を作ってから集客する**
```
❌ 商品作成 → 集客しようとする
✅ 集客 → 教育 → 販売（この順番が正解）
```

**パターン②: フォロワー＝見込み客だと思っている**
```
フォロワー: 「見ている人」
見込み客:   「買う気がある人」

この2つは違う。変換する工程が"教育"
```

**パターン③: AIをコンテンツ量産ツールにしている**
```
❌ AIでX投稿を100個作る
✅ AIで設計図（商品・LP・導線）を作る

使い方を知ってる人が、圧倒的に少ない
```

**パターン④: 良い商品を作れば売れると信じている**
```
売れる = 気づいてもらう → 興味 → 信頼 → 決断

どこかが抜けると売れない
→ この全工程を設計するのが「ローンチの設計図」
```

#### E-7: オートローンチシステム全体図（★最重要スライド）
- 型: フロー図（視覚的に一枚で完結させる）

```
【集客フェーズ】
X発信（教育コンテンツ）
  ↓
LINE/オプチャ登録
  ↓
チャレンジ（CL）への誘導

【教育フェーズ】
DAY1〜5 ライブセミナー
  ↓
ワーク + アウトプット共有

【販売フェーズ】
Day4 セールスパート
  ↓
個別面談（ローンチ設計セッション）
  ↓
クロージング
```

> **デザイン指針**: テキストは最小限。矢印とボックスで流れを視覚化。3フェーズは色分けする。

#### E-8: 「あなたはどこで詰まっているか」
- 型: 問いかけスライド

```
あなたのローンチ、どのフェーズが弱い？

□ 集客できていない（見てくれる人がいない）
□ 教育できていない（信頼が積み上がっていない）
□ 販売できていない（買ってもらえない）

チャットに「集客」「教育」「販売」のどれか送ってください
```

> ※ チャットを拾いながら「この分布、実はローンチ設計がない人の典型パターンです」とつなげる

#### E-9: AIが設計図を高速化する
- 型: 対応表（工程 × AI活用）

| 工程 | AIでできること |
|------|--------------|
| 集客 | X教育コンテンツの設計・生成 |
| 誘導 | LINE配信文・LP文章の生成 |
| 教育 | セミナー台本の骨格設計 |
| 販売 | セールス構造の最適化 |

> 「明日はこの上2つを実際に体験します」

#### E-10: Day3プレビュー（期待感最大化）
- 型: 次回内容のティザー

```
Day3でやること（明日）

① あなたの商品コンセプトをAIで生成する
   → プロンプトをコピペするだけ。30秒で出力される

② LP文章のドラフトをAIで作る
   → 骨格が出来上がる

"なんとなく分かった"が"実際に動いた"になる
```

## ■ Day2特有の設計ポイント

### 1. アウトプット紹介は "承認 + 代弁" の二役を担う
単なる紹介にせず、「なぜこれが商品になるか」を必ず一言添える。
参加者の代わりに「自分の当たり前が価値になる」という事実を証明する場として使う。

### 2. 振り返りは「覚えてる？」ではなく「思い出させる」設計
Day1の内容を問うのではなく、こちらから提示して「そうそう」を引き出す。
スライドにDay1の核心をテキストで出す（思い出させる）だけでOK。

### 3. オートローンチシステム全体図は1枚で完結させる
テキストで説明するより、図で見せる方が圧倒的に頭に入る。
この図が「設計図を持っていなかった人」にとって最も刺さるスライド。
デザインに一番時間をかけていい。

### 4. Day3への橋渡しは「体験できる」を具体的に見せる
「明日は実践します」だけでは弱い。
「プロンプトをコピペするだけ、30秒で出力される」という具体性が期待感を作る。
